# SMS Bot v6 授权系统 — 旧版 Workers 部署教程

> 当前 Windows 客户端已正式对齐到 SMSBOT-Auth-Server（https://license.918883.com）协议。
> 本文档保留为旧版 Cloudflare Workers 授权方案备份，不再作为当前正式接入说明。
> 如果你在维护现网，请以 SMSBOT-Auth-Server 服务端协议和 /api/client/* 接口为准。

---

## 你需要准备什么

在开始之前，确认你有以下东西：

| 需要的东西 | 说明 | 费用 |
|---|---|---|
| 一台 Linux 服务器 | 已有的就行，1核1G足够 | 你已有 |
| 服务器能上网 | 能访问 Telegram 和 Cloudflare | - |
| 一个邮箱 | 注册 Cloudflare 用 | 免费 |
| Telegram 账号 | 你已有 | 免费 |

总额外费用：**0 元**

## 第一部分：注册 Cloudflare 账号

### 1.1 打开注册页面

在浏览器打开：https://dash.cloudflare.com/sign-up

### 1.2 填写信息

- 输入你的邮箱
- 设置密码
- 点击「Create Account」

### 1.3 邮箱验证

去邮箱收一封验证邮件，点击里面的链接。

### 1.4 跳过域名设置

注册后会提示你添加域名，**直接跳过**，我们用不到域名。

✅ 完成！你现在有一个 Cloudflare 账号了。

## 第二部分：在 Linux 服务器上安装工具

用 SSH 登录你的 Linux 服务器，然后依次执行以下命令。

### 2.1 安装 Node.js

Node.js 是运行 Cloudflare 工具需要的。

```bash
# 检查是否已安装
node --version
```

如果显示 `v18.x.x` 或更高，跳到 2.2。

如果没有安装或版本太低：

```bash
# Ubuntu / Debian 系统
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo bash -
sudo apt-get install -y nodejs

# CentOS / Rocky Linux 系统
curl -fsSL https://rpm.nodesource.com/setup_20.x | sudo bash -
sudo yum install -y nodejs
```

安装完验证：

```bash
node --version
# 应该显示 v20.x.x
```

### 2.2 安装 Wrangler

Wrangler 是 Cloudflare 的命令行工具，用来部署代码。

```bash
npm install -g wrangler
```

安装完验证：

```bash
wrangler --version
# 应该显示 wrangler x.x.x
```

### 2.3 登录 Cloudflare

```bash
wrangler login
```

执行后会输出一个网址，类似：

```
Open a link in your browser: https://dash.cloudflare.com/oauth2/auth?...
```

**复制这个网址**，粘贴到你电脑的浏览器里打开。

页面会显示「Allow Wrangler to make changes to your Cloudflare account?」

点击「Allow」。

回到终端，应该显示：

```
Successfully logged in.
```

### 2.4 安装 Python 依赖

管理 Bot 需要 Python 3 和两个库。

```bash
# 检查 Python
python3 --version
# 应该显示 Python 3.8+

# 安装依赖
pip3 install python-telegram-bot httpx
```

### 2.5 上传文件

把 `sms_bot_v6.zip` 里的 `license-server` 文件夹上传到服务器。

方法一（推荐）：用 SFTP 工具（如 WinSCP、FileZilla）上传

方法二：命令行

```bash
# 在服务器上创建目录
mkdir -p /opt/license-server

# 在你的电脑上执行（替换你的服务器 IP）
scp -r license-server/* root@你的服务器IP:/opt/license-server/
```

上传完后，服务器上应该有这些文件：

```
/opt/license-server/
├── admin_bot.py          ← 管理 Bot 主程序
├── worker.js             ← 验证 API 代码
├── wrangler.toml         ← 部署配置
├── config.json.example   ← 配置模板
├── requirements.txt      ← Python 依赖
└── 部署指南.md
```

✅ 完成！工具准备好了。

## 第三部分：部署验证 API 到 Cloudflare

这是最核心的一步。验证 API 部署到 Cloudflare 的全球节点上，客户的 Bot 连这里验证，你的服务器 IP 完全隐藏。

### 3.1 进入目录

```bash
cd /opt/license-server
```

### 3.2 创建 KV 存储

KV 是 Cloudflare 的云端数据库，用来存授权记录。

```bash
wrangler kv:namespace create LICENSE_KV
```

执行后会输出类似：

```
🌀 Creating namespace with title "smsbot-license-LICENSE_KV"
✨ Success!
Add the following to your configuration file in your kv_namespaces array:
{ binding = "LICENSE_KV", id = "abc123def456ghi789jkl012" }
```

**重要：复制那个 id 值**（`abc123def456ghi789jkl012` 那一串）。

### 3.3 修改配置文件

```bash
nano wrangler.toml
```

找到最后一行：

```
id = "在这里填你的KV_NAMESPACE_ID"
```

把 `在这里填你的KV_NAMESPACE_ID` 替换成你刚才复制的 id。

改完后应该像这样：

```
id = "abc123def456ghi789jkl012"
```

按 `Ctrl+O` 保存，`Ctrl+X` 退出。

### 3.4 设置管理密码

这个密码用于保护管理接口，只有你的管理 Bot 知道这个密码。

```bash
wrangler secret put ADMIN_SECRET
```

提示输入时，**输入一个复杂密码**，例如：

```
Xk9pM2n7H4j6Lw8Q
```

⚠️ **记住这个密码！** 后面配置管理 Bot 要用。

密码不会显示在屏幕上，输完直接回车。

### 3.5 部署

```bash
wrangler deploy
```

第一次部署可能会提示你选择 Cloudflare 账号，直接回车选默认的。

成功后会显示：

```
Published smsbot-license (0.50 sec)
  https://smsbot-license.你的用户名.workers.dev
```

⚠️ **记下这个网址！** 这就是你的验证 API 地址。

### 3.6 测试 API

```bash
curl "https://smsbot-license.你的用户名.workers.dev/verify?machine_id=TEST-0000-0000-0000"
```

应该返回（首次访问自动试用）：

```json
{"valid":true,"expires":"2026-03-23","trial":true,"trial_hours":24,"msg":"首次使用，已自动激活 24 小时免费试用"}
```

如果看到这个返回，说明 API 部署成功！

✅ 完成！验证 API 已上线。

## 第四部分：部署授权管理 Bot

这个 Bot 运行在你的服务器上，你通过 Telegram 发命令来管理客户授权。

### 4.1 创建管理 Bot

打开 Telegram，搜索 `@BotFather`，发送以下消息：

```
/newbot
```

BotFather 会问你：

> Alright, a new bot. How are we going to call it? Please choose a name for your bot.

输入一个名字，例如：

```
我的授权管理
```

然后 BotFather 会问：

> Good. Now let's choose a username for your bot.

输入一个用户名（必须以 bot 结尾），例如：

```
my_license_admin_bot
```

BotFather 会回复你一个 Token，类似：

```
7123456789:AAHxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

**复制这个 Token。**

### 4.2 获取你的 User ID

在 Telegram 搜索 `@userinfobot`，发送任意消息。

它会回复你的信息，找到 `Id:` 后面的数字，例如：

```
Id: 123456789
```

**记下这个数字。**

### 4.3 创建配置文件

```bash
cd /opt/license-server
cp config.json.example config.json
nano config.json
```

填入你的信息：

```json
{
    "bot_token": "7123456789:AAHxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx",
    "admin_user_id": 123456789,
    "worker_url": "https://smsbot-license.你的用户名.workers.dev",
    "admin_secret": "Xk9pM2n7H4j6Lw8Q"
}
```

各字段说明：

| 字段 | 填什么 |
|---|---|
| bot_token | 步骤 4.1 BotFather 给你的 Token |
| admin_user_id | 步骤 4.2 获取的数字 ID |
| worker_url | 步骤 3.5 部署后得到的网址 |
| admin_secret | 步骤 3.4 你设置的密码 |

按 `Ctrl+O` 保存，`Ctrl+X` 退出。

### 4.4 测试运行

```bash
cd /opt/license-server
python3 admin_bot.py
```

如果显示：

```
2026-03-22 12:00:00 [INFO] 授权管理 Bot 已启动
```

打开 Telegram，找到你创建的管理 Bot，发送：

```
/start
```

应该收到管理中心的欢迎消息。

如果一切正常，按 `Ctrl+C` 停止，进入下一步设置后台运行。

### 4.5 设置后台运行（systemd）

```bash
# 创建服务文件
sudo nano /etc/systemd/system/license-bot.service
```

粘贴以下内容（注意替换路径和用户名）：

```ini
[Unit]
Description=SMS Bot License Manager
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/opt/license-server
ExecStart=/usr/bin/python3 /opt/license-server/admin_bot.py
Restart=always
RestartSec=5
Environment=PYTHONUNBUFFERED=1

[Install]
WantedBy=multi-user.target
```

> 如果你不是用 root 登录的，把 `User=root` 改成你的用户名。

按 `Ctrl+O` 保存，`Ctrl+X` 退出。

```bash
# 启用并启动
sudo systemctl daemon-reload
sudo systemctl enable license-bot
sudo systemctl start license-bot

# 检查状态
sudo systemctl status license-bot
```

应该显示 `Active: active (running)`。

### 4.6 查看日志（排错用）

```bash
# 实时查看日志
sudo journalctl -u license-bot -f

# 或者查看程序日志
tail -f /opt/license-server/license_bot.log
```

✅ 完成！管理 Bot 已在后台运行。

## 常见问题

### Q：客户换了电脑怎么办？

新电脑机器码不同，会自动获得新的 24 小时试用。客户把新机器码发给你，你重新授权即可。旧机器的授权不用管，不影响。

### Q：我想改试用时长怎么办？

打开 `license-server/worker.js`，搜索 `24`，把 `24 * 60 * 60 * 1000` 改成你想要的小时数。改完执行 `wrangler deploy` 重新部署。

### Q：客户说验证失败怎么办？

1. 让客户发 `/machine_id` 获取机器码
2. 你用 `/check 机器码` 查一下状态
3. 如果没授权 → `/license 机器码 天数 客户名`
4. 如果已过期 → `/extend 机器码 天数`
5. 让客户发 `/activate` 重新验证

### Q：Cloudflare Workers 免费够用吗？

每天 10 万次请求，每个客户每天大约请求 4-10 次（启动 + 定期检查）。你需要 1 万个客户才能用完免费额度。

### Q：管理 Bot 挂了怎么办？

不影响客户使用。客户的验证走的是 Cloudflare Workers（永远在线），管理 Bot 只是你操作授权用的。重启就行：

```bash
sudo systemctl restart license-bot
```

### Q：怎么备份授权数据？

数据存在 Cloudflare KV 里（云端），不会丢失。你也可以定期用 `/licenses` 截图保存。

### Q：怎么完全删除一个客户的记录？

`/revoke 机器码` 会删除 KV 里的记录。下次这台机器连验证 API 时，会重新获得 24 小时试用。
